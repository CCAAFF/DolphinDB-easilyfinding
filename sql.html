表 1 · 谓词速查

谓词(predicate)是返回布尔值的表达式,主要用在 where / having / on 子句中做行级判断。下表按类别系统归纳,并补充易错细节(通配符规则、子查询谓词语义、空值短路等)。

类别谓词作用与要点示例
比较谓词= / ==等于。与 NULL 比较结果为 NULL(非 true),空值判断须用 is null。where price = 50
!= / <>不等于。where sym != `AAPL
> / >=大于 / 大于等于。where price >= 100
< / <=小于 / 小于等于。where qty < 200
between … and范围包含(闭区间,含两端)。where price between 10 and 50
notBetween … and范围排除(开区间外)。where price notBetween 10 and 50
集合谓词in值属于给定集合 / 向量 / 单列子查询结果。where sym in [`AAPL,`GOOG]
where sym in (exec sym from t2)
notIn值不属于给定集合。where sym notIn [`AAPL,`GOOG]
模式匹配谓词like字符串模式匹配。? 匹配单个字符,% 匹配零到多个字符。where sym like "?A%"
notLike字符串模式排除。where sym notLike "?A%"
空值谓词is null判断是否为空值。空值判断必须用此谓词,不能用 = NULL。where price is null
is not null判断是否非空。where price is not null
子查询谓词exists子查询存在性判断:子查询至少返回一行则为 true。相关子查询可引用外层表列。where exists (select * from t2 where t2.sym=t1.sym)
not exists子查询不存在性判断:子查询无行返回则为 true。where not exists (select * from t2 where t2.sym=t1.sym)
any与单列子查询比较,任意一个满足即 true(配合 > < = 等比较符)。where price > any (exec price from t2)
all与单列子查询比较,全部满足才 true。where price > all (exec price from t2)
逻辑组合and / or / not谓词逻辑组合。注:where 中用 , 连接=从左到右层层过滤(顺序影响性能);用 and 连接=各自基于原表过滤再取交集(与序列相关时必须用 and)。where price > 50 and qty < 200

表 2 · 数据来源模块

select / exec / from 等核心查询入口，决定数据从哪来、以什么形式返回。

关键字作用简述例子/备注
select核心查询，返回表。select * from t1
exec返回向量/矩阵，不以表形式输出。exec price from t1
from指定数据来源，是 select / exec 的搭配部分。select * from t1
partition指定分区查询，减少扫描范围。select * from t1 where partition(sym, 0 1)
map分区独立执行再合并，适合分区间独立计算。select * from t1 map
as重命名。select price as openprice from t1

表 3 · WHERE 过滤条件模块

行级过滤条件，支持范围、集合、模式匹配、空值判断和子查询比较。

关键字作用简述例子/备注
where基础行过滤。select * from t1 where price > 50
between范围包含。select * from t1 where price between 10 and 50
notBetween范围排除。select * from t1 where price notBetween 10 and 50
in集合包含。select * from t1 where sym in [`AAPL,`GOOG]
notIn集合排除。select * from t1 where sym notIn [`AAPL,`GOOG]
like字符串模式匹配（? 代表一个未知字符、% 代表零到多个字符）。select * from t1 where sym like "?A%"
notLike字符串模式排除。select * from t1 where sym notLike "?A%"
is null判断空值。select * from t1 where price is null
exists子查询存在性判断。select * from t1 where exists (select * from t2 where t2.sym=t1.sym)
any与子查询比较，任意一个满足。select * from t1 where price > any (exec price from t2)
all与子查询比较，全部满足。select * from t1 where price > all (exec price from t2)

表 4 · GROUP 分组与聚合模块

分组聚合与窗口分析，context by 保留原始行数适合窗口类计算。

关键字作用简述例子/备注
group by常规分组聚合。select avg(price) as avgPrice from t1 group by sym
context by分组但保留原始行数，适合窗口类计算。select sym, price, mavg(price,3) as mp from t1 context by sym
cgroup by累计分组，对有序数据做累计聚合。select cumsum(qty) as cumQty from t1 cgroup by sym
having聚合后再过滤。select sym, avg(price) as avgPrice from t1 group by sym having avg(price) > 50
interval按时间区间分组。select sum(qty) from t1 group by interval(date, 1d)
over窗口/分析函数。select sym, price, rank(price) over(partition by sym) as rk from t1

表 9 · JOIN 表连接模块

DolphinDB 丰富的连接族：内/左/全/交叉/等值/时序/窗口/前缀/半连接等。

关键字作用简述例子/备注
inner join / ej内连接，返回两表中匹配的行。select * from t1 inner join t2 on t1.sym=t2.sym
select * from ej(t1, t2, `sym)
lj / left join左连接，保留左表所有行。select * from lj(t1, t2, `sym)
fj / full join全连接，保留两表所有行。select * from fj(t1, t2, `sym)
cj / cross join笛卡尔积连接。select * from cj(t1, t2)
ej / equi join等值连接，按等值条件匹配。select * from t1 equi join t2 on t1.sym=t2.sym
select * from ej(t1, t2, `sym)
aj / asof join常用于最近时间戳向前匹配，时序专用。select * from t1 asof join t2 on t1.sym=t2.sym
select * from aj(t1, t2, `sym`date)
wj / pwj用于时间窗口内的关联和聚合连接。select * from wj(t1, t2, 0:5, <[sum(qty)]>, `sym`date)
pj / prefix join前缀匹配连接。select * from pj(t1, t2, `sym)
lsj左半连接，只保留左表中有匹配的行。select * from lsj(t1, t2, `sym)
sej等值半连接，强调按等值条件匹配。select * from sej(t1, t2, `sym)
right join右连接，保留右表所有行。select * from t1 right join t2 on t1.sym=t2.sym

表 11 · SQL 执行顺序

DolphinDB SQL 各子句的真实执行顺序(共 11 步)。重点:使用 context by 时,csort 与 limit/top 的顺序与不带 context by 时不同 —— 这是写时序因子最容易踩坑的地方。

步骤子句说明 / 注意点
1from最先执行,包含 from 中的表对象及 lj / ej / aj 等表连接,以及本子句中的子查询。
2on应用连接条件,只保留符合条件的行。外连接(lj/right 等)中未匹配的行以空值补齐保留。
3where舍弃不满足条件的行。不能引用 select 里的新列名(as 别名、计算列)。, 连接=从左到右层层过滤;and 连接=各自基于原表过滤再取交集。
4group by / context by / pivot by对过滤后的行按指定列分组。
5csort仅配合 context by。对每个组内的行排序,直接影响 mavg/cumsum/deltas 等时序函数的结果。
6having对分组后的结果过滤(配合 group by / context by)。同样不能引用 select 别名。
7select此时才计算 select 子句中的表达式。
8distinct去除重复记录。
9limit / top
(有 context by 时)		应用于每个组:n 组 + limit m → 最多 n×m 行。此时执行顺序在 order by 之前。select top 2 * from t context by sym csort ts desc
10order by在 select 之后执行,因此可引用 select 别名 / 计算列。
11limit / top
(无 context by 时)		应用于全体结果,执行顺序在 order by 之后,丢弃范围外的行。

特殊:cgroup by — 先过滤 → 按 cgroup by + group by 列分组计算 select 项 → 按 order by(必须用,且须属于分组列)排序 → 最后算累计值。可用 [HINT_EXPLAIN] 查看执行计划验证是否走了优化路径。

表 12 · HINT 关键字(调优)

HINT 是 SQL 标准之外的扩展,用于直接指导优化器的执行策略,在特定场景下显著提升性能。写法:紧跟在 select / exec 后,用方括号包裹,如 select [HINT_EXPLAIN] * from t。以下为官方支持的全部 10 个 HINT。

关键字flag作用说明示例
HINT_LOCAL1分布式集群下,仅获取本地所有节点数据的查询结果。select [HINT_LOCAL] sum(*) from pt
HINT_HASH32group by 分组时优先采用哈希算法。select [HINT_HASH] count(*) from t group by sym
HINT_SNAPSHOT64用 registerSnapshotEngine 注册快照引擎后,从分布式表的快照引擎查询数据。select [HINT_SNAPSHOT] * from loadTable(db,tb)
HINT_KEEPORDER128确保 context by 分组后的输出顺序与输入保持一致。select [HINT_KEEPORDER] cumsum(vol) from t context by date, sym
HINT_SEQ512内存紧张时使分区查询串行执行,节约并发资源开销。timer select [HINT_SEQ] avg(vol) from t
HINT_NOMERGE1024跳过 map 中间结果的合并步骤,直接返回分区表的句柄而非合并后的内存表。select [HINT_NOMERGE] price from pt context by ticker
HINT_PRELOAD4096仅 TSDB 引擎支持。where 条件过滤前先把相关数据列加载到内存再过滤。select [HINT_PRELOAD] sum(price) from t where volume > 100000
HINT_EXPLAIN32768打印查询执行计划,用于实时监测查询速度与执行顺序,SQL 性能调优首选。select [HINT_EXPLAIN] * from tb where id > 20
HINT_SORT524288group by 分组时使用排序算法处理数据。select [HINT_SORT] avg(price) from trades group by sym
HINT_VECTORIZED4194304group by 分组时采用向量化运算(vectorization)。select [HINT_VECTORIZED] sum(price) from trades group by sym
