一、流数据 流式发布订阅:21 个流表操作 + 流式 SQL

1.1 · 21 个流表操作

流式数据类是特殊的内存表，用于发布订阅。流表只能往里面追加（Append）数据，不能修改或删除历史记录。按「组别」归类如下。

组别对应函数作用内容
建表与共享streamTable, keyedStreamTable, haStreamTable, share创建内存/带主键/高可用流表，以及将表共享为全局名称供所有会话访问。
写入与连接辅助appendMsg, appendForJoin向流表追加消息；为连接引擎左右表写入数据，用于流表连接场景。
订阅消费subscribeTable, unsubscribeTable建立或取消对流表的订阅关系，将新数据推送到处理函数或目标表。
持久化与缓存enableTablePersistence, disableTablePersistence, clearTablePersistence, enableTableCachePurge, enableTableShareAndCachePurge, enableTableShareAndPersistence控制流表落盘、关闭/清除持久化、内存缓存自动清理，以及「共享+缓存清理 / 共享+持久化」组合。
元信息与运维existsStreamTable, existsSubscriptionTopic, getStreamTables, dropStreamTable, removeTopicOffset查看流表/订阅是否存在、获取全部流表信息、删除流表及订阅、删除指定 topic 的 offset 记录。
流表属性控制setStreamTableFilterColumn, setStreamTableTimestamp设置流表的过滤列（订阅时按列值过滤）和时间戳列（用于时间对齐与回放控制）。

1.2 · 流式 SQL

把一条 SQL 变成持续运行、持续出结果的实时任务：先把表「登记」为数据源，再注册查询，最后像订阅流表一样订阅结果。

函数作用简述
declareStreamingSQLTable将指定表声明为流式 SQL 输入表；只有声明过的表才能用于注册流式 SQL 查询，且不影响该表的普通 SQL 使用。
registerStreamingSQL注册一个流式 SQL 查询并返回 queryId，自动创建结果变更日志流表；支持 SELECT / WHERE / 部分 JOIN / ORDER BY。
revokeStreamingSQL撤销已注册的流式 SQL 查询。
revokeStreamingSQLTable取消某张表作为流式 SQL 输入表的声明。
subscribeStreamingSQL订阅某个流式 SQL 查询的实时输出结果或结果变更。
unsubscribeStreamingSQL取消对某个流式 SQL 查询结果的订阅。
getStreamingSQLStatus查询流式 SQL 任务状态，支持查看单个查询或全部查询；管理员可查看所有用户的查询。
listStreamingSQLTables列出当前用户声明过的流式 SQL 输入表；管理员可查看全部声明信息。
getStreamingSQLSubscriptionInfo查询流式 SQL 的订阅信息（被谁订阅、订阅情况如何）。
